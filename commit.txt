Fix Frobenius: FAIL in rebis translate command

rebis/translate.rs, run_pipeline(): the Frobenius check (μ∘δ=id) did a
Protein→mRNA→Protein roundtrip by calling translate(&back_mrna), which
scans for an AUG start codon. reverse_translate picks the first codon from
codons_for_aa per AA; for Met that is AUC (not AUG — the index/cache
encoding has T/F swapped for codon positions 2 and 3, making AUC sort first
as a Met codon). translate("AUC...") finds no AUG anywhere → returns an
empty protein → Frobenius always FAIL for every sequence.

Fix: replaced the translate(&back_mrna) call with a direct codon-by-codon
loop using translate_codon(). No AUG scanner needed for a structural
roundtrip. codons_for_aa() returns exactly the codons where
translate_codon(c)==aa holds, so translating back codon-by-codon is
always identity → Frobenius: PASS.

Dynamic sequence builder — kernel builds its own IMASM sequence from IgTuple

New module src/sequence.rs:
- aggregate_votes(IgTuple) → Scores: each of the 12 IG primitives casts
  weighted votes for the IMASM token it structurally prefers (e.g. T_net
  votes FSPLIT×2, Phi_c votes ENGAGR×2 + EVALT×1 + EVALF×1, H2 votes
  IMSCRIB×2, K_trap votes IFIX×2)
- build_next_program(tuple, len, self_ref): assembles a Program from the
  vote ordering, enforcing: (1) never reduce stack below 0, (2) FSPLIT
  balanced by FFUSE before end, (3) IMSCRIB bookends when self_ref, (4)
  length from tier (O_0=6, O_1=8, O_2=10, O_∞=12)
- vote_summary() display: top-5 voted tokens with scores

Kernel modifications (kernel.rs):
- Kernel.dynamic_mode: bool field (default false)
- try_self_modify(): when dynamic_mode, derive IgTuple from dynamic_imscribe(),
  call build_next_program(), replace program in place; otherwise old overflow
  logic unchanged
- load_dynamic(): enable dynamic mode, build first sequence from current snapshot
- disable_dynamic(): disable without changing program

main.rs / menu.rs:
- mod sequence added
- 'dynamic' command: enable/disable dynamic mode, show program + vote summary
- 'dynamic' entry in PROGRAMS_MENU

Builds clean. No preset sequences needed in dynamic mode — the kernel
discovers its own execution path from its current structural state.
