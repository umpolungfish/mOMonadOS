Add "ruleset dialetheic" — Belnap-FFUSE bridge for cross-universe verdicts (2026-06-16)

New REPL command: `ruleset dialetheic <catalog_name> <8|9>`. Computes the
entry's two real, already-established verdicts — F under canonical (U0,
ordinal-correct gate check) and T under the closing universe (U8 or U9,
gate + T_CEILING) — feeds them into the kernel's actual ParaVM as r1=T,
r2=F, runs the real FFUSE instruction (Belnap join, not a shortcut call),
and reports the fused r0.

Live result, booted in QEMU:
  `ruleset dialetheic birch_swinnerton_dyer 8` -> canonical=F, U8=T,
    FFUSE -> B (designated, dialetheic)
  `ruleset dialetheic hodge_conjecture 9` -> canonical=F, U9=T,
    FFUSE -> B (designated, dialetheic)

This does NOT make BSD/Hodge true under canonical — B is "designated but
contradictory," not T. The honest claim is narrower and still real:
canonical's flat-F verdict is in conflict with genuine T-evidence from
another universe, and the kernel's own Belnap algebra (already used
elsewhere for RH's critical line, Majorana/SIC-POVM fixed points) treats
that conflict as B, not as F. Refactored t_ceiling_check into a silent
core (t_ceiling_check_silent) shared by the printing version and this
new command.

Separately confirmed: the kernel's existing `psm frob` (ENGAGR->FSPLIT->
FFUSE->HALT on its own isolated ParaVM) already demonstrates the same
absorb/split/fuse/retain cycle in general — r0: N -> B (engaged) ->
split to T,F -> fused back to B. That VM is independent of the Kernel
struct used by ruleset verify/compute_tier; this command is the first
thing that deliberately seeds real catalog-derived verdicts into it.
