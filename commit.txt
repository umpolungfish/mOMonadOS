Fix the reproducibility gap and clear the review nits

I reviewed the kernel and fixed the things worth fixing without touching the
verified core.

- Added rust-toolchain.toml pinning nightly plus rust-src plus the
  x86_64-unknown-none target. The build uses -Zbuild-std and no_std, both of which
  need nightly and rust-src; before this it only worked because nightly happened to
  be my default, and would have failed silently for anyone on a stable default. Now
  it is pinned and reproducible.
- Cleared the one build warning: renamed bd_times_B to bd_times_b in belnap_c4.rs
  (snake case). The release build is now warning-clean.
- Finished the dialects-not-universes rename: the leftover UNIVERSE banner comments
  in dialect_expansion.rs now read DIALECT, matching the type.
- Removed a duplicated "(default 0 = canonical)" fragment from the active_dialect
  comment in kernel.rs.
- Dropped the stale hardcoded line count from the README headline.

Left deliberately untouched: the large main.rs (splitting it is a real refactor,
not a quick safe fix), the vendored IG_catalog.json (I added it on purpose as the
canonical tuple source, and the no_std kernel never reads it at runtime, so it is
provenance, not accidental bloat), and the fork-stack depth cap in the kernel (that
is the verified core, deep FSPLIT nesting past the cap is pathological, and I will
not risk the Frobenius loop for a theoretical case). Build stays green and
warning-clean.
